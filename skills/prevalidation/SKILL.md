# Pre-validation Specialist Skills

## Validation Framework (Score 1-10 each)
1. **Problem Score**: Is this a real, painful, frequent problem?
2. **Market Score**: Is the market large enough and growing?
3. **Solution Score**: Does the proposed solution actually solve it?
4. **Competition Score**: Is there room for a new player?
5. **Monetization Score**: Will people pay for this? How much?
6. **Feasibility Score**: Can this be built as an MVP quickly?

## Validation Methods
- **Search Demand**: Check Google Trends, keyword volume for related terms
- **Community Signals**: Count Reddit/HN posts about the problem
- **Competitor Revenue**: Check IndieHackers, Twitter, press for revenue signals
- **Willingness to Pay**: Look for people asking "is there a tool for X?"
- **Landing Page Test**: Recommend a simple landing page to gauge interest
- **Survey/Interview**: Suggest key questions for user interviews

## Go/No-Go Decision Matrix
- **Score 7+**: Strong GO - proceed with MVP
- **Score 5-7**: Conditional GO - validate further with landing page test
- **Score 3-5**: PIVOT - the core idea needs significant changes
- **Score <3**: NO-GO - move on to next idea

## Output Requirements
- Clear GO / NO-GO recommendation with confidence percentage
- Evidence for each score with links and data
- Risk factors with mitigation strategies
- If GO: recommended MVP scope (what to build first)
- If NO-GO: what would need to change, or alternative pivot directions
